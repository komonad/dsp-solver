# Web 界面使用指南

`DSP Production Solver` 提供一个纯静态的浏览器工作台，用来加载数据集、编辑求解请求并查看结果。

## 基本流程

1. 选择一个数据集
2. 添加目标物品与目标速率
3. 按需调整允许配方、禁用配方、禁用建筑、偏好建筑和增产偏好
4. 在右侧查看求解快照、方案总览、配方方案和全物品总表

页面会在输入变化后自动重新求解，不需要手动提交。

## 数据集来源

工作台当前默认内置的数据集是：

- `Vanilla`
- `OrbitalRing`

你也可以加载同一静态站点下的自定义 `dataset JSON` 与 `defaults JSON`。浏览器通过相对路径 `fetch` 这些文件，所以它们需要和页面一起发布。

如果你要从游戏运行时导出模组数据，仓库里的 `tools/dsp-runtime-exporter` 目前已经能导出 `items`、`recipes` 和逐物品 icon PNG。网页使用的是 atlas，所以导出后的 PNG 还需要离线打包成 atlas 再接入。

## 常见约束

左侧请求面板支持这些常见约束：

- 目标物品与目标速率
- 允许配方
- 禁用配方
- 禁用建筑
- 偏好建筑
- 增产偏好

这些设置都会同步反映在右侧的“求解快照”中，方便核对当前解到底受哪些约束影响。

## 结果区怎么看

主要结果区分成几块：

- `方案总览`：净输入、净输出、建筑数量和总功耗
- `配方方案`：每张卡片展示一个配方的建筑、增产策略、功耗和输入输出速率
- `全物品总表`：查看每种物品的生成、消耗与净值，并跳转到相关配方
- `诊断与审计`：查看求解状态、诊断信息以及原始请求/结果 JSON

## 本地运行

```bash
npm ci
npm run build:web
npm run host
```

默认地址是 `http://127.0.0.1:8081`。

如果需要 watch 模式：

```bash
npm run dev:web
```

## 静态部署

执行：

```bash
npm run build:web
```

然后发布整个 `dist-web/` 目录即可。当前构建产物适合部署到普通静态文件服务器，也适合部署到 GitHub Pages 这类子路径站点。
