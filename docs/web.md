# Web 界面使用指南

DSP 计算器提供一个基于浏览器的可视化界面，支持实时计算和交互式配置。

## 功能特性

### 🎯 需求管理
- 添加多个产物需求（如 60/分钟处理器 + 30/分钟量子芯片）
- 设置现有产线供给（如已有 50/分钟铁板供给）
- 标记物品为原矿（作为外部输入，不参与生产计算）

### 🏭 建筑选择

支持两种级别的建筑选择：

**全局类别覆盖**
- 按建筑类别（熔炉/制造台/化工厂/研究站）统一设置默认建筑
- 优先级：全局选择 > 特定配方选择 > 默认建筑

**单配方选择**
- 为特定配方选择具体建筑型号
- 展开配方详情后可见建筑下拉菜单

### ⚡ 增产剂配置

增产剂设置采用**全局覆盖**模式：

**全局增产剂**
- 设置全局增产剂后，自动应用于所有支持增产的配方
- 仅对 `proliferatorLevel > 0` 且建筑有增产槽位的配方生效
- 不支持的配方自动使用无增产模式

**单配方自定义**
- 全局未设置时，可为特定配方单独配置增产剂
- 勾选"自定义增产剂"后选择等级和模式

**加速 vs 增产模式**

| 模式 | 效果 | 适用场景 |
|------|------|----------|
| 加速 | 建筑数量减少，产出不变 | 节省建筑、降低功耗 |
| 增产 | 产出增加，建筑数量不变 | 提高产能 |

### 📊 结果展示

**配方卡片**
- 显示总建筑数和等效建筑数
- 点击展开查看子配方详情
- 显示实际建筑数量（含小数）和向上取整数量

**物料平衡**
- 原矿需求列表
- 中间产物结余/缺口
- 电力需求估算

**配方详情**
- 输入/输出物品速率
- 实际建筑占用率
- 单配方功率计算

## 数据格式

Web 界面使用 `data/*.json` 文件作为数据源，兼容 [dsp-calc](https://github.com/DSPCalculator/dsp-calc) 格式。

### 加载自定义数据

将自定义数据文件放入 `data/` 目录，格式如下：

```json
{
  "items": [...],
  "recipes": [...]
}
```

刷新页面后会自动加载可用的数据文件。

## 状态保存

配置自动保存到浏览器 localStorage：
- 需求列表
- 增产剂设置（全局 + 单配方）
- 建筑选择（全局 + 单配方）
- 原矿标记

清除浏览器数据会重置所有配置。
