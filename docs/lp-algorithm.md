# 线性规划算法详解

## 问题建模

戴森球计划的生产计算可以建模为**线性规划（Linear Programming）**问题。

### 变量定义

```
x_i = 配方 i 的执行次数（每分钟）
```

每个配方对应一个变量，表示该配方每分钟执行多少次。

### 目标函数

默认目标：最小化总建筑数量

```
min Σ(time_i / 60) × x_i
```

其中 `time_i` 是配方的制作时间（秒）。

### 约束条件

**1. 需求约束（目标产物必须满足需求）**

```
Σ(output_ij × x_i) >= demand_j
```

对于每个需求物品 j，所有配方产出的总和必须大于等于需求。

**2. 非原矿约束（中间产物不能出现缺口）**

```
Σ(output_ik × x_i) >= 0
```

对于非原矿物品 k，净产出必须非负（产出 >= 消耗）。

**3. 非负约束**

```
x_i >= 0
```

所有配方执行次数必须非负。

## 多产物配方配平

### 问题描述

原油精炼配方：`2原油 → 1精炼油 + 2氢气`

如果只需要精炼油，会产生多余的氢气。LP 求解器会自动：

1. 寻找消耗氢气的下游配方（如液氢燃料棒）
2. 计算最优比例，使氢气刚好被完全消耗
3. 确保精炼油产出满足需求

### 配平示例

假设需要 180 精炼油/分钟：

```
配方A：2原油 → 1精炼油 + 2氢气（需要 180 次）
配方B：1氢气 → 1液氢燃料棒（消耗氢气）
```

LP 求解器输出：

```
x_A = 180（原油精炼执行 180 次/分钟）
x_B = 360（液氢燃料棒执行 360 次/分钟）
```

验证：
- 精炼油产出：180 × 1 = 180 ✓
- 氢气净产出：180 × 2 - 360 × 1 = 0 ✓

## 求解器实现

使用 [yalps](https://github.com/Ivoks/yalps) 纯 JavaScript LP 求解器。

### 求解步骤

1. **构建模型**：将生产需求转换为 LP 标准形式
2. **添加约束**：需求约束 + 物料平衡约束 + 非负约束
3. **求解**：使用单纯形法求解最优解
4. **验证**：检查解的可行性和物料平衡

### 无解情况处理

当 LP 问题无可行解时（如需求不可能满足），求解器会返回：

```typescript
{
  feasible: false,
  reason: 'DEMAND_NOT_SATISFIABLE',
  details: {...}
}
```

常见原因：
- 循环依赖（A 需要 B，B 需要 A，没有外部输入）
- 配方不支持（没有可用的生产配方）
- 原矿标记错误（需要的物品被标记为原矿但外部输入不足）

## 优化目标

支持多种优化目标：

### min-buildings（默认）

最小化总建筑数量，适合资源充足的情况。

```
min Σ buildings_i
```

### min-waste

最小化副产物浪费，适合不希望处理副产物的情况。

```
min Σ waste_j
```

### min-power

最小化总功耗。

```
min Σ power_i × buildings_i
```

## 增产剂处理

增产剂效果作为参数乘数应用到模型中：

**加速模式**：
- 建筑需求减少：`buildings_i = base_buildings / (1 + speedBonus)`

**增产模式**：
- 单配方产出增加：`output = base_output × (1 + productivityBonus)`
- 需要的配方次数减少

## 复杂度分析

- 变量数：配方数量（通常 100-500）
- 约束数：物品数量 + 需求数量（通常 200-1000）
- 求解时间：毫秒级（单纯形法）

对于戴森球计划规模的数据集，LP 求解可以在瞬间完成。
